Agente 3 — The Memory

Guia passo a passo para configurar e rodar o projeto do Agente de Negociação com Memória

O que é este Agente?

Este projeto demonstra como construir um Agente de IA Stateful (com memória) para simular a negociação de financiamento automotivo.

O cenário: um cliente quer financiar um SUV elétrico de R$ 350.000. O agente propõe taxas de juros, o cliente pode recusar, e após um número configurável de recusas o agente faz handoff para um atendente humano (circuit breaker).

Conceitos-chave

  • Google ADK (Agent Development Kit) com Gemini 2.0 Flash
  • Máquina de Estados Finita (FSM) — controla o funil de negociação com estados tipados via Pydantic
  • Memória de Curto Prazo — checkpoints de sessão (Session Service)
  • Memória de Longo Prazo — busca vetorial RAG (Vector Search)
  • FinOps — comparação de custo: abordagem "amnésica" vs. "stateful"
  • Graceful Degradation — quando serviços externos falham, o agente continua com mocks
  • Retry com Exponential Backoff via tenacity

Arquitetura

┌─────────────────────┐ │ main.py │ ← Orquestrador └────────┬────────────┘ │ ┌────────▼────────────┐ │ StatefulFinance │ ← agent_router.py │ Agent (Router) │ LLM + Gateways └──┬──────────┬───────┘ │ │ ┌────────────▼──┐ ┌───▼──────────────┐ │ Session GW │ │ Memory GW │ │ (Curto Prazo) │ │ (Longo Prazo/RAG) │ │ session_gw.py │ │ memory_gw.py │ └───────┬───────┘ └───────┬───────────┘ │ │ ┌────────────▼──┐ ┌────────▼────────────┐ │ Vertex Session│ │ Vertex Vector Search │ │ Service / Mock│ │ / Mock │ └───────────────┘ └─────────────────────┘

Passo a Passo

1

Clonar o Repositório

Abra o terminal e execute:

git clone https://github.com/glauciodaniel/agent-3-the-memory.git
cd agent-3-the-memory
2

Criar o arquivo .env

O arquivo .env guarda as credenciais e configurações sensíveis do projeto. Ele não vai para o Git (está no .gitignore), então você precisa criá-lo manualmente na raiz do projeto.

Escolha uma das opções abaixo:

Mais simples

Opção A — Google AI Studio (API Key)

Ideal para dev local. Não precisa de projeto GCP. 

# Opção A: Google AI Studio
GOOGLE_API_KEY=sua-api-key-aqui

Gere sua API Key em aistudio.google.com/apikey

Opção B — Vertex AI (GCP)

Para quem já tem projeto no Google Cloud.

# Opção B: Vertex AI
GOOGLE_GENAI_USE_VERTEXAI=1
GOOGLE_CLOUD_PROJECT=seu-project-id
GOOGLE_CLOUD_LOCATION=us-east1
Atenção Você também precisa estar autenticado via gcloud auth application-default login.

Referência completa de variáveis

Variável Descrição Obrigatória?
GOOGLE_API_KEY API Key do Google AI Studio (Opção A) Sim, se não usar Vertex AI
GOOGLE_GENAI_USE_VERTEXAI Defina como 1 para usar Vertex AI (Opção B) Sim, se usar Vertex AI
GOOGLE_CLOUD_PROJECT ID do seu projeto no Google Cloud Sim, se usar Vertex AI
GOOGLE_CLOUD_LOCATION Região do Vertex AI (ex: us-east1) Sim, se usar Vertex AI
VECTOR_SEARCH_ENDPOINT_ID Endpoint de Vector Search. Se ausente, usa mock automático Não
USE_VERTEX_SESSION Defina como true para Session Service real. Se ausente, usa mock em memória Não
Dica Para rodar o projeto rapidamente, basta a Opção A com a GOOGLE_API_KEY. As variáveis opcionais usam mocks automaticamente quando não configuradas.
3

Criar ambiente virtual e instalar dependências

Crie um ambiente virtual Python para isolar as dependências do projeto.

Windows

python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt

macOS / Linux

python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
O que será instalado? google-adk (Agent Development Kit), google-cloud-aiplatform (Vertex AI), pydantic (validação de estado), rich (tabelas no terminal), Jinja2 (templates), tenacity (retry automático), tiktoken (contagem de tokens) e python-dotenv (carregamento do .env).
4

Rodar o Agente

Com o ambiente virtual ativado e o .env configurado: 

python -m src.main

O que esperar na saída

  1. O agente faz o contato inicial com o cliente
  2. Analisa o crédito e propõe uma taxa de juros
  3. O cliente recusa a proposta
  4. Após 3 recusas (configurável), o agente faz handoff para humano
  5. Ao final, um relatório FinOps compara custo "amnésico" vs. "stateful"
5

Rodar os Testes

Para validar que está tudo funcionando:

pytest tests/ -v

Os testes verificam: a máquina de estados (FSM), o circuit breaker, o controle de concorrência (OCC) e o graceful degradation da memória.

Alternativa: Rodar com Docker

Se preferir não instalar Python localmente, rode tudo via Docker. O docker-compose.yml carrega o .env automaticamente. 

# Construir a imagem e rodar
docker compose up --build
Importante O arquivo .env deve existir na raiz do projeto antes de rodar o docker compose.

Solução de Problemas

Erro Solução
ModuleNotFoundError: No module named 'google.adk' Verifique se o venv está ativado e se rodou pip install -r requirements.txt.
GOOGLE_API_KEY not found ou API key not valid Verifique se o .env existe e se a API Key está correta. Gere uma nova em aistudio.google.com/apikey.
Python version < 3.11 Python 3.11+ é obrigatório. Verifique com python --version.
FileNotFoundError: .env Crie o arquivo .env na raiz conforme o Passo 2.
docker: command not found Instale o Docker Desktop ou use a opção local (Passos 3–5).
PermissionError ao ativar venv no Windows PowerShell como Admin: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
Timeout / erro de conexão com Vertex AI Se não tem acesso ao Vertex AI, use a Opção A (API Key). Os mocks funcionam automaticamente.
